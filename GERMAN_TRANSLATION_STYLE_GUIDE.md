# KnitTools German Translation Style Guide

Last updated: 2026-05-09

This guide is the working reference for translating KnitTools pages, tools, and articles into German. Use it before translating any German content. The goal is fluent German that sounds natural to knitters, keeps the original meaning intact, and stays consistent across all 38 articles and the tool pages.

## Core Principles

- Preserve the original meaning. Do not add new advice, remove caveats, soften warnings, or change the level of certainty.
- Translate for German-speaking knitters, not word-for-word from English or Finnish.
- Prefer standard German knitting terms used in German-language patterns, yarn shops, and knitting blogs.
- Keep the tone practical, calm, and direct. The articles should feel like a knowledgeable knitter explaining the point clearly.
- Use `du` in articles and tools. It is natural for German knitting tutorials and keeps the tone approachable.
- Avoid clumsy literal translations. If the direct translation is not what knitters say, use the German knitting term.
- Keep KnitTools as the brand name. Do not translate it.
- Keep article style consistent with the English and Finnish versions: concise, useful, slightly opinionated, no decorative filler.

## German Voice

Use:

- `du`, `dein`, `dir`
- active sentences
- short paragraphs
- concrete knitting vocabulary
- everyday German verbs: `stricken`, `messen`, `anschlagen`, `abketten`, `aufnehmen`, `zunehmen`, `abnehmen`, `spannen`

Avoid:

- formal `Sie`
- marketing-heavy language
- overexplaining basic concepts when the source is concise
- machine-translated compounds
- English terms where German knitters normally use German terms
- gender-star forms in SEO titles and headings

For inclusivity, prefer neutral phrasing when it reads naturally:

- Good: `Wer mehrere Projekte parallel strickt, verliert schnell den Überblick.`
- Good: `Beim Stricken hilft ein Reihenzähler...`
- Use sparingly: `Strickerinnen und Stricker`
- Avoid in headings: `Stricker*innen`, `Anfänger*innen`

## Yarn Weight Categories

German does not have a fully native equivalent for the US yarn weight system. Keep the category names that German knitters encounter on yarn labels and in English-language patterns, but explain them in German.

Use:

| English category | German display |
|---|---|
| Lace | `Lace` |
| Super Fine / Fingering | `Fingering / Sockengarn` |
| Sport | `Sport` |
| DK / Light | `DK / Light` |
| Worsted / Medium | `Worsted / Aran / Medium` |
| Bulky | `Bulky / Chunky` |
| Super Bulky | `Super Bulky / Super Chunky` |
| Jumbo | `Jumbo` |

Guidelines:

- Use `Garnstärke` as the general concept.
- Keep `DK`, `Worsted`, `Aran`, `Fingering`, `Bulky`, and `Lace` when they are categories.
- Use `Sockengarn` when the article talks about typical sock yarn, but do not treat every fingering yarn as sock yarn.
- Explain WPI as `Wraps per Inch (WPI)` on first use, then `WPI`.
- For German readers, emphasize meters and 10 cm. Keep yards/inches only where the tool or source needs conversion.

## SEO And Frontmatter

German frontmatter should be written for German search behavior, not translated mechanically.

Rules:

- Keep titles clear and under normal SEO length where possible.
- Put the main German search term near the beginning.
- Use `Garn`, `Maschenprobe`, `Nadelstärke`, `Strickanleitung`, `Reihenzähler`, and `Strickrechner` consistently.
- Tags should be German phrases that users might search.
- Keep `translationKey` identical to the English source.
- Do not add a German article to any translation map until it is meant to be publicly linked.
- Draft unpublished German translations if the implementation mirrors the Finnish workflow.

Examples:

- `title: "Maschenprobe stricken: Schritt für Schritt"`
- `description: "So strickst, wäschst und misst du eine Maschenprobe richtig, damit Größe, Garnbedarf und Nadelstärke zur Anleitung passen."`
- `tags: ["Maschenprobe stricken", "Maschenprobe messen", "Nadelstärke anpassen", "Strickanleitung Maschenprobe"]`

## Units And Formatting

- Use metric first: `cm`, `m`, `g`, `mm`.
- Mention inches/yards only for conversion tools, US patterns, or when the source does.
- Write `10 x 10 cm` in plain Markdown unless the file already uses typographic multiplication signs.
- In prose, German decimal comma is natural: `2,5 mm`.
- In code/data values, keep machine-readable decimal points if the existing data does.
- Use `Meter` in prose and UI labels, `m` in compact tables.
- Use `Yards` for yards. Do not translate as `Jards`.
- Use `Zoll` in prose when explaining inches, but `in` can stay in compact labels like `4 in`.

## When To Keep English Terms

Keep or include the English term when German knitters commonly encounter it:

- `WPI`
- `DK`
- `Worsted`
- `Aran`
- `Fingering`
- `Lace`
- `Bulky`
- `Super Bulky`
- `Kitchener Stitch` only as secondary to `Maschenstich`
- `Ravelry`
- `PDF`
- `Android`
- `App`

For AI features, prefer German `KI` in German copy:

- `KI-Hilfe`
- `KI-gestützter Scanner`
- `KI-Zusammenfassung`

Do not translate brand or product names:

- `KnitTools`
- `Ravelry`
- Android platform names

## Common False Friends And Bad Translations

Avoid these:

- `Garngewicht` for `yarn weight` -> use `Garnstärke`
- `Messgerät` for calculator -> use `Rechner` or `Tool`
- `Spannung` for gauge -> use `Maschenprobe`; `Fadenspannung` only means tension
- `Strickmuster` for full knitting pattern -> use `Strickanleitung`
- `Nadelgröße` as the main term -> use `Nadelstärke`
- `Fadenlänge` for project yardage -> use `Garnbedarf` or `benötigte Meter`
- `einweben` for weaving in ends -> use `Fäden vernähen`
- `Blockieren` for blocking -> use `spannen`, `waschen und spannen`, or `in Form ziehen`
- `Doppelspitznadeln` for DPNs -> use `Nadelspiel`
- `Stockinette` in German article prose -> use `glatt rechts`

## Translation Checklist

Before translating:

- Read the English source and Finnish translation if one exists.
- Identify the article's core concept and choose the locked German term from this guide.
- Search the existing German files once German content exists, to avoid drift.
- Keep the same factual structure and caveats.

During translation:

- Use `du`.
- Keep terms consistent within the article.
- Do not introduce new knitting advice.
- Keep examples, numbers, and assumptions equivalent to the source.
- Convert idioms into natural German, not literal phrases.

After translation:

- Check title, description, tags, headings, FAQ questions, and internal links.
- Check every `gauge`, `yarn weight`, `needle size`, `cast on`, `bind off`, `row`, and `round` occurrence.
- Ensure article links point to the correct German route only when that route exists and is publishable.
- Run a build only after implementation work, not while drafting terminology alone.
